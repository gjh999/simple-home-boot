# Awesome eGovFrame (개발자가 직접 만들어 공유한 프로젝트) ![eGovFrame Curated](https://img.shields.io/badge/eGovFrame-Curated-3178c6)

> 전자정부 표준프레임워크(eGovFrame) 생태계의 오픈소스 프로젝트, 컴포넌트, 학습 자료를 정리한 목록

전자정부 표준프레임워크는 공공부문 정보화 사업의 표준 개발 프레임워크입니다.
이 저장소는 표준프레임워크를 기반으로 하거나 함께 활용할 수 있는 **커뮤니티 프로젝트**를
한 곳에 모아 소개합니다.

> 📢 **"직접 신규 개발 또는 대규모 개선한 유용한 소스코드 프로젝트"의 GitHub 레포지토리에 대해 PR 부탁드립니다.**

> 본 저장소를 포크 > README 사용자 공유 프로젝트 항목에 한 줄 추가 > Pull Request를 보내 주시면 검토 후 등재됩니다.

## ■ 이 저장소의 목적

- 개발자가 직접 만든 표준프레임워크 관련 프로젝트를 **누구나 쉽게 찾아 활용할 수 있도록** 정리합니다.
- 각 프로젝트는 기본적으로 **개발자가 직접 만들어 공유한 프로젝트** 입니다.
- 본 저장소는 링크만 모아 정리하며 의미 있다고 판단되는 프로젝트의 경우 표준프레임워크 공식 레포지토리로 포크하여 관리될 수 있습니다.
- 본 목록 등재가 **공식 인증을 의미하지 않습니다**. 사용 시 라이선스와
  유지보수 상태를 직접 확인해 주세요.

## 🌟 커뮤니티 기여 사례

표준프레임워크 생태계는 커뮤니티 개발자의 기여로 확장됩니다.
다음은 **개인 개발자의 프로젝트가 공식 저장소로 편입된 실제 사례**입니다.
처음 기여를 고민하는 분들이 흐름을 이해할 수 있도록 정리했습니다.

### 사례: Homebrew Tap (homebrew-egovframe)

표준프레임워크 개발환경을 macOS 에서 **Homebrew 명령어로 간편 설치**할 수 있게 해주는
Tap 저장소입니다.

#### 진행 과정

```
[1단계] 개인 저장소에서 자유롭게 개발
   github.com/leejongyoung/homebrew-egovframe
              │
              │ ① 개발자가 표준프레임워크 센터에 기여 의사 전달
              ▼
[2단계] 표준프레임워크 센터 검토
   - 코드 품질, 라이선스, 생태계 가치 평가
   - 원 개발자의 유지보수 의사 확인
              │
              │ ② 검토 통과
              ▼
[3단계] 공식 organization으로 Fork 편입
   github.com/eGovFramework/homebrew-egovframe
              │
              │ ③ 원 개발자가 희망하면 표준프레임워크 센터에서 메인테이너 권한 부여
              ▼
[4단계] 공식 도구로 사용자에게 제공
```

#### 관련 저장소

- **원 저장소 (개인 개발)**: [leejongyoung/homebrew-egovframe](https://github.com/leejongyoung/homebrew-egovframe)
- **공식 편입 저장소**: [eGovFramework/homebrew-egovframe](https://github.com/eGovFramework/homebrew-egovframe)
- **Homebrew 공식 저장소 (표준프레임워크 맥용 개발환경)**: [Homebrew/homebrew-cask — egovframedev.rb](https://github.com/Homebrew/homebrew-cask/blob/main/Casks/e/egovframedev.rb)

### 이 사례에서 시사 점

1. **개인 GitHub에서 자유롭게 시작하세요.**
   자신의 계정에서 마음껏 만들어 공개 하시면 됩니다.

2. **본 Awesome 리스트가 첫 번째 소통의 통로입니다.**
   여기 등재되면 다른 개발자 및 표준프레임워크 센터에서 프로젝트를 인지할 수 있고, 다양한 긍정적인 영향을 줄수 있습니다.
   등재 자체로 공식 인증은 아니지만, 노출이 시작됩니다.

3. **품질·필요성이 검증되면 공식 편입될 수 있습니다.**
   표준프레임워크 센터에서 검토 후 생태계에 가치 있다고 판단하면 공식 표준프레임워크 레포지토리로
   **Fork 방식**으로 편입합니다. 모든 기여가 편입되는 것은 아니며,
   편입 여부는 표준프레임워크센터에서 결정합니다.

4. **원 개발자의 기여는 유지되며, 희망하면 계속 참여할 수 있습니다.**
   공식 편입 후에도 원 개발자가 희망하면 표준프레임워크 센터에서 메인테이너 권한을
   부여해 계속 참여할 수 있도록 합니다.

## ■ 기여하기

이 목록에 프로젝트를 추가하고 싶으시다면 [CONTRIBUTING.md](CONTRIBUTING.md)를 참고하세요.

**기본 등재 기준**
- 공개 저장소(GitHub, GitLab 등)에 코드가 공개되어 있을 것
- 명확한 오픈소스 라이선스가 명시되어 있을 것 (Apache-2.0, MIT 등 권장)
- README에 프로젝트 설명, 사용법, 표준프레임워크와의 관계가 기술되어 있을 것
- 최근 1년 이내 커밋·이슈 응답 등 **유지보수 활동**이 확인될 것

**등재 방법**
1. 이 저장소를 포크합니다.
2. 해당 카테고리에 알파벳 순서로 한 줄 추가합니다.
   ```
   - [저장소명](URL) - 한 줄 설명 (마침표로 끝).
   ```
3. Pull Request를 보냅니다.
