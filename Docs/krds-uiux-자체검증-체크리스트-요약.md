# 디지털 정부서비스 UI/UX — 자체 검증 체크리스트 (요약본)

> 원문: 행정안전부 「디지털 정부서비스 UI/UX 가이드라인 자체 검증 체크리스트」 (`krds-uiux-자체검증-체크리스트.md`, 약 13,500줄).
> 본 문서는 **프런트엔드 전수검사용**으로 핵심 점검 항목을 추린 요약본이다. 정확한 준수 기준·예외·미준수 사례는 원문을 확인한다.

## 판정 기호
- **P (Pass)**: 준수  ·  **F (Fail)**: 미준수  ·  **E (Exception)**: 예외 처리  ·  **N/A**: 해당 없음
- 검증 단위: (서비스 유형 → 화면 → 요소). 동일 요소가 여러 화면에 반복되면 대표 화면으로 표집 가능.
- 우선순위: **스타일·컴포넌트의 '필수' 항목 우선** → 권장 → 우수.

---

## A. 스타일

### A-1. 색상
- [ ] Primary/Secondary/Gray/System **팔레트·색상값**을 그대로 사용한다.
- [ ] 컴포넌트별 **상호작용 상태(hover/active/focus/disabled)** 색·방식을 적용한다.
- [ ] 정의된 **배경 색상**을 사용하고, 배경·테두리·텍스트 색을 **일관**되게 표현한다.
- [ ] 시스템 색상은 **의미 체계(정보/성공/경고/위험)**를 따른다.

### A-2. 서체(타이포그래피)
- [ ] **Pretendard GOV**(또는 고딕 계열) 서체를 사용한다.
- [ ] 본문 기본 텍스트 **17px 이상**(자체 상징 16px 이상).
- [ ] 여러 줄 텍스트 **행간 = 글자 크기의 1.5배 이상**.
- [ ] 서체 **굵기 2종 이상** 구분 + 각 굵기에 **맞는 폰트 파일** 사용.
- [ ] **Title·Heading**을 본문보다 굵거나 크게, 서체 규칙을 일관 적용.

### A-3. 형태
- [ ] 정의된 형태(반경·테두리) 규격을 사용한다.
- [ ] 컨테이너 둥글기로 **내부 콘텐츠가 잘리지 않는다**.

---

## B. 컴포넌트

### B-1. 아이덴티티(공식 배너·운영기관 식별자·헤더·푸터)
- [ ] 공식 배너를 **모든 화면 최상단**에 두고 스타일·텍스트를 **변형하지 않는다**.
- [ ] 운영기관 식별자의 안내 텍스트·배치를 **일관**되게 제공한다.
- [ ] **푸터**: 모든 화면 동일 위치, **개인정보 처리방침**(전자상거래 시 이용약관) 표시, 자주 찾는 링크 배치, 최신 연락처 제공, 스타일 수정 최소화.
- [ ] **헤더**: 스타일 수정 최소화, **유틸리티 링크 4개 이하**, **아이콘 버튼/링크에 텍스트 레이블** 제공.

### B-2. 탐색
- [ ] **건너뛰기 링크(skip-nav)**를 모든 화면에 제공(단순 화면은 생략 가능), 이동 상호작용 구현.
- [ ] **메인 메뉴(GNB)**: 확장/축소 상호작용을 적절히 구현.
- [ ] **브레드크럼**: 메인/랜딩에서 미사용, **구분자 제공**, 본문 제목과의 사이에 다른 요소 없음, **본문 그리드 왼쪽 정렬**, 링크 간격 충분, GNB를 대체하지 않음.
- [ ] **사이드 메뉴**: 작은 화면에서 미표시, 탐색 상호작용 구현.
- [ ] **콘텐츠 내 탐색**: 제목·활성 인디케이터 제공, 텍스트가 섹션 제목과 일치, 작은 화면은 제목과 본문 사이 배치.
- [ ] **페이지네이션**: **전체 화면 수 표시**, 한 화면에 1개만, 화면 크기와 무관하게 개수 유지, 직접 이동 상호작용 구현.

### B-3. 액션(링크·버튼)
- [ ] 링크 레이블이 **연결 목적지를 정확히 반영**한다.
- [ ] 링크·버튼 탐색 상호작용(키보드 포커스 포함)을 적절히 구현한다.
- [ ] 버튼 **계층(primary/secondary/tertiary)**·크기를 의미에 맞게 사용한다.

### B-4. 선택(라디오·체크박스·셀렉트·태그)
- [ ] 다중 선택/옵션 과다 시 **라디오·셀렉트 미사용**(체크박스/검색형 사용).
- [ ] 라디오: 선택 되돌리기 제공. 셀렉트: **레이블·선택값** 제공.
- [ ] 체크박스: 단일 선택에 미사용, 전체선택 시 **중간 상태** 표현.
- [ ] 태그: 레이블 간결, 대화형/비대화형 혼용 금지, 삭제 상호작용 구현.

### B-5. 입력(텍스트·텍스트영역·날짜·파일)
- [ ] 모든 입력에 **레이블 제공**, **복사·붙여넣기 허용**.
- [ ] 텍스트 영역: 글자 수 제한 시 **글자 수 정보** 표시.
- [ ] 날짜: 상대 날짜에 미사용, 다중 입력에 '년/월/일' 레이블.
- [ ] 파일 업로드: **크기·형식 정보 표시**.

---

## C. 기본 패턴

### C-1. 입력폼
- [ ] 입력폼에 **제목** 제공, 폼·콘텐츠 사이 **충분한 간격**.
- [ ] 입력 컴포넌트 주변에 **레이블·설명**, 필요 시 **단위** 표시.
- [ ] **필수/선택 항목을 명확히 구분**.
- [ ] 제출 버튼에 **비활성화 상태** 활용(미충족 시).
- [ ] **모바일 고려** 디자인.

### C-2. 개인 식별 정보 입력
- [ ] 입력 용도를 명확히 설명, **기본값 미설정**, 복붙 허용.
- [ ] 이름: 단일 입력 필드·모든 문자 지원·형식 설명. 생년월일: 연/일에 셀렉트 미사용·형식 안내. 성별: **'선택 안 함'** 제공. 전화번호: 레이블·형식 안내·국가코드 콤보박스.

### C-3. 첨부파일 / 필터링·정렬
- [ ] 첨부파일 크기·형식 표시. 아이디/비밀번호 필드 복붙 허용.

---

## D. 서비스 패턴 (필수 항목 위주)
- 방문 / 검색(기능 찾기·검색어 입력·결과 확인·이용·재검색·종료) / 로그인(기능 찾기·방식 선택·정보 입력·이용) / 신청(대상 탐색→정보 확인→자격 확인→작성→확인·확정→완료→결과 확인) / 정책 정보 확인.
- 각 단계 사용성 가이드를 **필수 → 권장 → 우수** 순으로 점검.

---

## E. 접근성(공통, KWCAG/WCAG 2.1 AA)
- [ ] 키보드만으로 모든 기능 조작 가능 + **포커스 가시성(`:focus-visible`)**.
- [ ] 본문 텍스트/배경 **명도 대비 4.5:1 이상**(큰 텍스트 3:1).
- [ ] **색만으로 정보 전달 금지**(텍스트·아이콘 병행).
- [ ] 표 `caption`·`scope`, 폼 `label` 연결, 이미지 `alt`.
- [ ] 페이지 제목·랜드마크·heading 구조 적절.

---

## 검증 절차(요약)
1. **준비**: 대상 범위 표집, 검증 환경 통일(브라우저·해상도), 도구 준비(대비 검사기·접근성 검사기).
2. **수행**: 화면별로 위 항목을 P/F/E/N/A 판정. '준수 기준'의 검증 절차 순번대로 판단.
3. **결과서**: 미준수(F) 항목·미준수 사례·개선안 기록 → 품질 개선 반복.

> 자동 1차 점검 스크립트 예시는 `krds-conversion` 스킬의 `verify.sh` 참조.
